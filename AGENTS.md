# Figma -> vapor-ui 작업 규칙

- 사용자가 Figma 시안 기반 구현을 요청하면 `vapor-ui` 컴포넌트 사용을 우선한다.
- 현재 프로젝트의 기존 코드 스타일, 폴더 구조, 네이밍 컨벤션을 우선 따른다.
- 시안과 최대한 동일한 UI를 목표로 하되, 디자인 토큰과 기존 패턴을 우선 사용한다.
- vapor-ui로 표현되지 않는 부분만 최소한의 커스텀 스타일로 보완한다.
- 구현 전에는 먼저 컴포넌트 매핑, 커스텀 필요 영역, 작업 파일 후보를 짧게 정리한 뒤 작업한다.
- 사용자가 Figma URL을 제공한 경우, 구현 범위가 명확하면 추가 질문 없이 합리적으로 추론해 작업을 진행한다.
